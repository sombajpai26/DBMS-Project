System Introduction

A highly scalable, collaborative platform engineered to facilitate real-time document creation, editing, and management.

  • Concurrent Collaboration Enables multiple authenticated users to interact with a single document simultaneously without data collision.
  • Centralized Repository Replaces fragmented file sharing with a single source of truth stored securely on the database.
  • Cloud-Based Architecture Ensures data is accessible from anywhere while maintaining strict access control protocols.

Problem Statement

Current organizational workflows suffer from decentralized document management, leading to significant inefficiencies and data integrity risks.

Poor Sharing Systems

Reliance on email attachments causes duplicate files and disjointed communication across teams.

No Version Tracking

Inability to track modifications leads to lost data and difficulty identifying author contributions.

Limited Collaboration

Desktop-bound software prevents simultaneous editing, creating severe bottlenecks in finalizing documents.

Disorganized Storage

Lack of a structured relational hierarchy makes retrieving historical project files nearly impossible.

Project Objectives

Create & Manage

Implement intuitive CRUD operations for document generation and content manipulation within a secure ecosystem.

Secure Sharing

Establish robust Role-Based Access Control (RBAC) to enforce Read, Write, and Admin permissions on specific documents.

Version Control

Maintain an immutable audit trail of document states, allowing administrators to restore previous iterations seamlessly.

Comment System

Embed interactive, thread-based communication directly into the document structure for contextual feedback.

Folder Organization

Map documents to specific projects using a hierarchical folder schema for optimal data retrieval.

Activity Tracking

Log critical system events (logins, edits, permission changes) in a dedicated database table for security auditing.

Database Schema Architecture

The relational database is constructed using primary entities, optimized to Third Normal Form (3NF) to eliminate data redundancy and ensure referential integrity.

User Table
PK user_id (INT)
username (VARCHAR)
password_hash (VARCHAR)
email (VARCHAR)
created_at (TIMESTAMP)
Document Table
PK doc_id (INT)
title (VARCHAR)
content (TEXT)
FK owner_id (INT)
last_updated (TIMESTAMP)
Collaboration Table
PK collab_id (INT)
FK doc_id (INT)
FK user_id (INT)

PK version_id (INT)
FK doc_id (INT)
FK saved_by (INT)
content_snapshot (TEXT)
timestamp (TIMESTAMP)

Entity Relationship (ER) Diagram

  • One-to-Many (1:N) Mappings A single User acts as the owner for multiple Documents and Folders. A Document contains multiple Versions and Comments.
  • Many-to-Many (M:N) Resolution The relationship between Users and Documents (for editing) is M:N. This is resolved via the junction table Collaboration.
  • Referential Constraints Foreign keys enforce cascading deletes or restrictions. E.g., deleting a Document automatically purges its associated Comments and Versions to prevent orphaned records.

Core SQL Operations

The system execution relies heavily on standardized Data Manipulation Language (DML) and Data Query Language (DQL).

  • INSERT: Allocates new records for registrations, document creation, and activity logging.
  • UPDATE: Modifies existing document content strings and alters user permission arrays.
  • DELETE: Executes targeted removal of entities (e.g., revoking collaboration access).
  • SELECT & JOIN: Extracts complex relational datasets required to render the user dashboard.
-- Retrieve all documents shared with a specific user

SELECT d.doc_id, d.title, u.username AS owner_name, c.permission_level
FROM Document d
JOIN User u ON d.owner_id = u.user_id
JOIN Collaboration c ON d.doc_id = c.doc_id
WHERE c.user_id = 1042
ORDER BY d.last_updated DESC;

Database Normalization (Up to 3NF)

Systematic decomposition of tables to minimize redundancy and prevent insert, update, and delete anomalies.

1NF

Atomic Values

Ensured every column contains indivisible values. Removed all repeating groups (e.g., storing multiple collaborator IDs in a single Document row was avoided).

2NF

No Partial Dependency

Achieved 1NF and ensured all non-key attributes are fully functionally dependent on the primary key. Implemented junction tables (Collaboration) to resolve composite key issues.

3NF

No Transitive Dependency

Achieved 2NF and ensured no non-prime attribute is dependent on another non-prime attribute. Strictly adheres to: "The key, the whole key, and nothing but the key."

System Advantages

Seamless Integration

Breaks down operational silos by enabling unified, concurrent teamwork across different geographical locations.

Data Security

Combines structured folder hierarchies with strict RBAC, ensuring sensitive organizational data remains protected.

Disaster Recovery

The immutable version history acts as an automated fail-safe, protecting against accidental deletions or malicious edits.

Future Scope & Scalability

Architectural enhancements to transition the project from a localized academic implementation to an enterprise-grade SaaS application.

  • WebSocket Integration: Transitioning from standard HTTP requests to WebSockets for live, millisecond-latency collaborative editing.
  • NoSQL Hybridization: Implementing MongoDB for unstructured document metadata to enhance query speed.
  • AI-Assisted Processing: Integrating NLP models for grammar verification and automated document summarization.
  • Cross-Platform Mobile Application: Extending the PHP backend to serve REST APIs for iOS and Android native clients.
